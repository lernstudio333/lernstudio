# Integration API — Overview

The Lernstudio Integration API lets external tools (Google Docs sidebars, Confluence plugins, etc.) push flashcard content into the app without a user being present.

> **OpenAPI spec** — A formal OpenAPI 3.0 specification is planned once the API stabilises. The endpoint docs are structured to map directly to OpenAPI (one file per endpoint, consistent field tables, explicit request/response shapes), so the conversion will be mechanical when the time comes.

## Authentication — Two-token model

The API uses two different credentials depending on the endpoint.

### 1. Pairing code (bootstrap only)

A short-lived, one-time 6-digit code generated by a logged-in admin inside the app. It is used exactly once to exchange for a session token. It expires after **5 minutes**.

Obtain it via the **Connect Sources** modal in the avatar menu.

### 2. Session token (ongoing use)

A long-lived 64-character hex token tied to a specific connected document. Passed as a standard Bearer token on every subsequent request.

```
Authorization: Bearer <session_token>
```

The session token is issued by `POST /integration/sessions` and stored in the `integrations` table. It does **not** use Supabase JWT auth — it is validated by the edge function directly against the database.

---

## Endpoints

| Method | Path | Auth | Purpose |
|--------|------|------|---------|
| `POST` | [`/integration/pairing-codes`](./integration/pairing-codes.md) | Supabase JWT (admin) | Generate a pairing code |
| `POST` | [`/integration/sessions`](./integration/sessions.md) | Pairing code in body | Exchange code → session token |
| `POST` | [`/integration/cards`](./integration/cards.md) | Bearer session token | Push cards (with dry-run support) |

---

## Conventions

- All request and response bodies are **JSON**.
- Errors return `{ "error": "<message>" }` with an appropriate HTTP status code.
- `dry_run: true` on the cards endpoint previews changes without writing to the database. The response shape is identical whether dry-run or not.
- Card positions are **integers**. If omitted on insert, the card is appended after the current highest position in the target lesson.
